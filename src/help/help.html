¿Qué es WebMatic?

WebMatic es una extensión de navegador que graba y reproduce macros web: automatiza clics, escritura, navegación entre páginas y extracción de datos en cualquier sitio web, sin escribir código.

Compatible con Chrome, Edge y Firefox, en Windows, macOS y Linux.

¿Para qué sirve?

  • Rellenar formularios repetitivos de forma automática.
  • Extraer datos de páginas web a variables.
  • Navegar y hacer clic en secuencias complejas con lógica condicional.
  • Ejecutar la misma secuencia N veces con el modo bucle.
  • Reutilizar macros desde otras macros con call_macro.
⏺

Grabar una macro

  1. Abre la extensión haciendo clic en el icono WebMatic en la barra del navegador.
  2. Pulsa el botón ⏺ Grabar en la pestaña superior del panel lateral.
  3. El panel se minimiza y aparece una barra roja en la parte superior. Ahora todo lo que hagas queda grabado: navegación, clics, escritura, scroll, etc.
  4. Cuando termines, pulsa ⏹ Detener en la barra flotante.
  5. Escribe un nombre para la macro y pulsa Guardar.
💡 Consejo

El grabador registra pausas como pasos wait. Si una pausa dura más del umbral configurado (por defecto 3 s), se inserta un paso de espera. Ajústalo en Configurar → Umbral de pausa.

¿Qué se graba automáticamente?

  • Clics y dobles clics en cualquier elemento.
  • Texto escrito en campos de entrada, textareas y elementos editables.
  • Navegación a otras URLs.
  • Marcado / desmarcado de checkboxes y radio buttons.
  • Scroll hacia elementos (cuando supera un umbral de desplazamiento).
  • Hover sobre elementos con tooltips.
▶

Reproducir y controles

  1. Selecciona una macro de la lista en la pestaña Reproducir.
  2. Si necesitas precargar datos, abre el bloque Datos de autocompletado (opcional) (aparece colapsado por defecto para dejar más espacio a la lista de macros).
  3. Pulsa ▶ Reproducir para ejecutarla una vez.
  4. Para repetirla varias veces, configura el número de repeticiones y pulsa ▶▶ Bucle.
  5. Mientras se ejecuta, un panel flotante muestra el paso actual y el progreso.
  6. Pulsa ⏹ Detener en cualquier momento para cancelarla.

Datos de autocompletado (colapsable)

El bloque de datos en Reproducir se muestra colapsado por defecto. Si no lo usas, ocupa solo una línea y deja más espacio visible para las macros.

  • Activa Usar dato en esta ejecución para habilitar la precarga.
  • Selecciona tipo y valor del dato principal (por ejemplo: DNI, N° de afiliado, autorización, etc.).
  • Con + Agregar otro dato puedes cargar múltiples valores en una misma ejecución.

Plantillas de datos

Puedes guardar combinaciones de datos para reutilizarlas cuando quieras.

  1. Configura tu combinación de datos (dato principal + adicionales).
  2. Pulsa Guardar actual y asigna un nombre.
  3. Luego, selecciona la plantilla y pulsa Aplicar.
  4. Si ya no la necesitas, usa Quitar plantilla.

Controles de velocidad

En Configurar → Velocidad de reproducción ajusta el multiplicador de velocidad entre 0.5× (más lento) y 3× (más rápido). A velocidad 1× se respetan los tiempos originales grabados.

Botón +1s aquí

Si la reproducción se atasca en un paso concreto (por ejemplo, la página carga lento), pulsa el botón flotante ⏱ +1s aquí para agregar 1 segundo de espera extra antes de ese paso en la macro guardada.

💡 Consejo

Si la macro contiene un paso navigate, la reproducción se reanuda automáticamente en la nueva página después de cargarla.

Cada paso de una macro representa una acción específica. Puedes ver y editar los pasos desde el Editor visual.

⚙

Control de flujo

Estos pasos especiales permiten tomar decisiones, repetir acciones e iterar sobre datos. Se agregan desde el Editor visual → Agregar paso.

⊙ if_exists — Si existe

Ejecuta un bloque de pasos then si el selector existe en la página, o un bloque else si no existe. Útil para manejar elementos opcionales como modales, banners de cookies o mensajes de error.

↺ loop_until — Repetir hasta

Repite un bloque de pasos mientras un selector no existe (o sí existe, según configures). Ideal para esperar a que un spinner desaparezca o a que aparezca un botón "Siguiente". Configura el máximo de iteraciones para evitar bucles infinitos.

⚡ try_fallback — Intentar / Alternativa

Intenta ejecutar un bloque de pasos (try). Si algún paso falla, ejecuta el bloque alternativo (fallback) en lugar de detener la macro. Perfecto para manejar variaciones de UI entre sesiones.

▦ for_each_row — Por cada fila

Itera sobre las filas de una tabla HTML. En cada iteración, los valores de las columnas indicadas se cargan como variables {{COL1}}, {{COL2}}, etc., y se ejecutan los pasos del bloque.

💡 Consejo

Los bloques de control de flujo pueden anidarse: puedes tener un if_exists dentro de un loop_until, o un for_each_row que llame a otra macro con call_macro.

📦

Variables

Las variables almacenan valores que puedes reutilizar en cualquier campo de texto usando la sintaxis {{NOMBRE_VARIABLE}}.

¿Cómo usar una variable?

Escribe {{NOMBRE}} en cualquier campo de un paso. Al reproducir la macro, el valor se sustituye automáticamente. Ejemplo: en un paso input, pon el valor Hola {{NOMBRE}}, tu código es {{CODIGO}}.

extract
Captura el texto visible de un elemento de la página y lo guarda en la variable que indiques.
set_variable
Asigna un valor fijo a una variable en tiempo de grabación. Útil para constantes o valores calculados.
prompt
Muestra un cuadro de diálogo al usuario durante la reproducción y guarda lo que escriba en una variable.
for_each_row
Carga las columnas de cada fila como variables. Si defines columnas NOMBRE, EMAIL, tendrás {{NOMBRE}} y {{EMAIL}} disponibles en los pasos del bloque.
💡 Ejemplo práctico

Graba una macro que haga extract del nombre de un usuario en {{USUARIO}}, luego usa input con valor Bienvenido, {{USUARIO}} para rellenar un campo de saludo automáticamente.

✏

Editor visual de pasos

El editor visual te permite ver, agregar, reordenar y eliminar los pasos de cualquier macro sin tocar el script IIM.

  1. Selecciona una macro de la lista y pulsa el icono de edición (✎) que aparece al lado.
  2. Se abre el Script Editor con dos pestañas: Visual y Script IIM.
  3. En la pestaña Visual, cada paso aparece como una tarjeta con su tipo, selector y descripción.
  4. Usa los botones ↑ ↓ para reordenar pasos, ✎ para editar y 🗑 para eliminar.
  5. Pulsa + Agregar paso para insertar un nuevo paso al final.
  6. Cuando termines, pulsa 💾 Guardar para persistir los cambios.

Pestaña Script IIM

Cambia a la pestaña Script IIM para editar el script de texto directamente. El formato es compatible con iMacros. Puedes pegar scripts externos, editarlos y guardarlos. Los cambios en Script IIM se reflejan en la vista Visual al guardar.

💡 Consejo

Usa Guardar como… para crear una copia de la macro con un nombre diferente, manteniendo la original intacta.

💾

Exportar / Importar

Accede a todas las opciones de exportación en la pestaña Configurar → Exportación y respaldo.

⬇ Exportar macro como .iim

Selecciona una macro y pulsa ⬇ Exportar seleccionada (.iim). Se descarga un archivo de texto compatible con el formato iMacros, que puedes compartir o importar en otros navegadores.

💿 Backup de todas las macros

Pulsa Backup de todas las macros para descargar un archivo .json con todas tus macros. Guárdalo como copia de seguridad.

📥 Importar backup (.json)

Pulsa Importar backup (.json) y selecciona el archivo de backup. Las macros se cargarán y fusionarán con las existentes.

⚠ Importante

Las macros se guardan en el almacenamiento local del navegador. Si desinstales la extensión o limpias los datos del sitio, las macros se perderán. Haz backups regularmente.

💡

Consejos y atajos

  • ⚡Velocidad óptima: para sitios lentos usa 0.5× o 0.75×. Para formularios rápidos usa 2× o 3×.
  • 🔄Modo bucle: configura el número de repeticiones con el campo numérico antes de pulsar ▶▶ Bucle. Se puede cambiar entre reproducciones.
  • 🎯Selectores robustos: el grabador genera selectores CSS automáticamente, pero si una página cambia su estructura, edita el selector en el editor visual para que sea más estable (usa id en lugar de clases genéricas).
  • 🧩Macros modulares: crea macros pequeñas y reutilizables, y combínalas con call_macro. Por ejemplo: una macro "Login" y otra "Procesar pedido" que llama a "Login" primero.
  • 🔍Buscar macro: usa la barra de búsqueda en la pestaña Reproducir para filtrar macros por nombre. La búsqueda es instantánea.
  • 📐Panel izquierda / derecha: cambia el lado del panel en Configurar → Visualización según la distribución de la página que estés automatizando.
  • 🌙Modo oscuro y temas: activa el modo oscuro y elige entre 4 variantes de color en Configurar → Temas. La preferencia se guarda automáticamente.
  • ⏱Umbral de pausa: si no quieres que las pausas naturales al grabar generen pasos wait innecesarios, sube el umbral a 5–8 s en Configurar → Umbral de pausa.
  • 🔁Reintentos con try_fallback: usa try_fallback para que la macro no se detenga ante pequeñas variaciones de la UI, como un botón que a veces aparece con texto diferente.
  • 📊Iterar tablas: usa for_each_row para procesar todas las filas de una tabla HTML automáticamente, sin grabar un paso por cada fila.
